# หมูกระทะ ระบบสั่งอาหาร (Node.js)

ระบบสั่งอาหารหน้าร้าน แปลงจากอาร์ติแฟกต์ React มาเป็นแอป **Node.js + Express** ที่รันได้จริงบนเครื่องหรือเซิร์ฟเวอร์ของคุณเอง พร้อมธีมร้านอาหารสีเขียว-เหลืองทันสมัย และปรับหน้าจอให้ใช้งานบนมือถือได้พอดี

## วิธีติดตั้งและรัน

```bash
npm install
npm start
```

จากนั้นเปิดเบราว์เซอร์ไปที่ **http://localhost:3000**

(ถ้าต้องการรันบนพอร์ตอื่น ตั้งค่า `PORT=8080 npm start`)

## โครงสร้างโปรเจกต์

```
moo-krata-node/
├── server.js          ← เซิร์ฟเวอร์ Express + REST API
├── data/               ← ไฟล์เก็บข้อมูล (เมนู/ผู้ใช้/ออเดอร์) เป็น JSON บนดิสก์
│   ├── menu.json
│   ├── users.json
│   └── orders.json
└── public/             ← หน้าเว็บฝั่งลูกค้า/พนักงาน (HTML+CSS+JS ล้วน ไม่ต้อง build)
    ├── index.html
    ├── css/style.css
    └── js/app.js
```

ข้อมูลทั้งหมดเก็บเป็นไฟล์ JSON ในโฟลเดอร์ `data/` แทนที่ `window.storage` ของอาร์ติแฟกต์เดิม จึงคงอยู่ถาวรตราบใดที่ไฟล์เหล่านี้ไม่ถูกลบ — ไม่ต้องพึ่งฐานข้อมูลภายนอก

## มุมมองในแอป

- **หน้าลูกค้า** (`?table=1..4` ต่อท้าย URL เพื่อล็อกโต๊ะสำหรับสแกน QR) — เลือกเมนู เพิ่ม/ลดจำนวน กดส่งออเดอร์ ดูสถานะออเดอร์ของตัวเองแบบเรียลไทม์
- **หน้าพนักงาน** — ล็อกอิน (ทดสอบ: `owner` / `owner123`) แล้วเข้าสู่:
  - **ครัว** — ดูตั๋วออเดอร์ เลื่อนสถานะ รอคิว → กำลังทำ → เสิร์ฟแล้ว
  - **เช็คบิล** (ผู้จัดการ/เจ้าของ) — รวมบิลต่อโต๊ะ พิมพ์สลิป ปิดบิล
  - **จัดการเมนู** (ผู้จัดการ/เจ้าของ) — เพิ่ม/แก้ไข/ลบเมนู
  - **จัดการผู้ใช้งาน** (เจ้าของ) — เพิ่ม/ลบพนักงาน กำหนดสิทธิ์
- **QR โต๊ะ** — สร้าง QR โค้ดสำหรับแปะแต่ละโต๊ะ ลิงก์ไปหน้าลูกค้าพร้อมล็อกเลขโต๊ะ

## หมายเหตุด้านความปลอดภัย

ระบบล็อกอินนี้เหมาะสำหรับต้นแบบ/ใช้งานภายในร้านเท่านั้น รหัสผ่านเก็บเป็นข้อความธรรมดาในไฟล์ `data/users.json` และไม่มีการเข้ารหัส session หากจะนำไปใช้งานจริงที่เปิดสู่อินเทอร์เน็ตสาธารณะ ควรเพิ่ม:
- การเข้ารหัสรหัสผ่าน (เช่น bcrypt)
- ระบบ session/JWT ที่ปลอดภัย
- HTTPS และการจำกัดสิทธิ์การเข้าถึง API
